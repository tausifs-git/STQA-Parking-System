## B) Defects List


### **Defect ID: BUG-01**

* **Test case:** `TC-05`
* **Class.Method:** `Wallet(double)`
* **Description:** The system allows Wallet to be initialized with a negative amount.
* **Suggested Fix:** A condition should check if `balance < 0` and throw an `invalid amount exception`.


### **Defect ID: BUG-02**

* **Test case:** `TC-19`
* **Class.Method:** `deductFunds(double)`
* **Description:** Wallet was initialized with a big number `10e9` and then a very big number was transferred `10e9 - 10e-9`. In this case, the remaining amount should be `10e-9`. But when the system deducted the amount remaining balance was `0`.
* **Suggested Fix:** Use `BigDecimal` instead of `double` to store balance.


### **Defect ID: BUG-03**

* **Test case:** `TC-31`
* **Class.Method:** `transferFunds(Wallet toWallet, double amount)`
* **Description:** During transfer of funds, fund was transferred to an null object (replicating a scenario of failed transfer). Runtime exception was given but fund was deducted from the original wallet.
* **Suggested Fix:** Before deducting fund, first check if the transfer was done successfully to a valid user object.


### **Defect ID: BUG-04**

* **Test case:** `TC-35` & `TC-84`
* **Class.Method:** `isCompatible(VehicleType, LocalDateTime, LocalDateTime)`
* **Description:** `isCompatible()` always returns false for vehicle Type `TRUCK`.
* **Suggested Fix:** A condition should allow truck to be compatible
  `case TRUCK:
    if (slotType == ParkingSlotType.LARGE) {
           return isAvailable(startTime, endTime);
    }
    break;
  `

### **Defect ID: BUG-05**

* **Test case:**
* **Class.Method:** `isCompatible(VehicleType, LocalDateTime, LocalDateTime)`
* **Description:** Inside the `isCompatible()` function a `break` is missing inside `case MICROCAR`.
* **Suggested Fix:** A `break` should be added to the `case`,
  `         case MICROCAR:
                if (slotType == ParkingSlotType.COMPACT || slotType == ParkingSlotType.REGULAR) {
                    return isAvailable(startTime, endTime);
                }
                break;
            default:
                return false;`



### **Defect ID: BUG-06**

* **Test case:** `TC-36`
* **Class.Method:** `isCompatible`, `isAvailable`
* **Description:** If a slot is deactivated then that slot can not be neither compatible nor available. System correctly rejected `isCompatible` but failed to reject `isAvailable`.
* **Suggested Fix:** There should be a condition in `isAvailable()`:-
  `if (!isActive) {
       return false;
   }`
